1.1

What is an LLM?（什麼是大型語言模型？）

input：一串文字（前文）→LLM：深度神經網路transformer 架構 · 數百億個可調權重→output：「下一個 token 是誰」
的機率分佈

一句話定義：LLM 是一個被設計來理解、生成並回應類人文字的深度神經網路，在極大量的文字資料上訓練——有時涵蓋公開網路的一大部分。「large」是雙關：參數規模（動輒數百億個訓練中不斷調整的權重）與資料規模都空前地大。

而它學的任務樸素得驚人：next-word prediction——給前文、猜下一個字。這個任務刻意利用了語言天生的序列性，逼模型學會文字裡的脈絡、結構與關係。任務如此簡單、模型卻如此能幹，連許多研究者都感到意外——1.6 會回來拆解這件事。

🔰「理解」的免責聲明

本書說 LLM「理解」語言，指的是它能生成連貫、切題的文字，不代表它擁有人類式的意識或理解。另外，因為 LLM 會「生成」，它也被歸類為generative AI（GenAI）的一種。

1.3

Stages of building and using LLMs（打造與使用 LLM 的階段）

能用 ChatGPT，為什麼還要自己造一個？書中給的理由很實際：

  • 理解機制與極限的最好方式：從零寫一遍，你才知道它哪裡強、哪裡會壞。這套知識也是預訓練／微調現成開源模型的必修課。
  • 客製模型可以贏過通用模型：為特定領域打造的 LLM——如金融的 BloombergGPT、醫療問答模型——在自己的領域可勝過 ChatGPT 這類通才。
  • 隱私與自主：敏感資料不必交給第三方；小型客製模型還能直接跑在筆電、手機上（Apple 正在探索的方向），降低延遲與伺服器成本，更新節奏完全自己掌控。
工具說明：現今多數 LLM 用 PyTorch 深度學習框架實作，本書也是（附錄 A 有 PyTorch 速成）。

兩階段配方：pretraining → fine-tuning

打造 LLM 的通用流程只有兩大步。「pre」的意思是「先」——先在大而雜的資料上訓練出通用語言能力，再精修。點流程中任一節點看說明：

自監督：標籤就藏在文字裡

1.4

Introducing the transformer architecture（transformer 架構）

input：2017 論文
"Attention Is All You Need"→transformer＝encoder ＋ decoder核心零件：self-attention（第 3 章手刻）→output：兩條後代路線
BERT（encoder）／ GPT（decoder）

現代 LLM 幾乎都站在 transformer 上。它出自 2017 年的論文《Attention Is All You Need》，原本是為機器翻譯設計的（英→德、英→法），由兩個子模組組成：encoder 把輸入句子編碼成一串「捕捉了上下文資訊」的向量；decoder 讀著這些向量，一次一個字地生成輸出。

靈魂零件是 self-attention：讓序列中的 token 彼此衡量重要性，模型因此能在預測時「選擇性注意」輸入的不同部分、抓住長距離的依賴關係。機制細節書中刻意先不展開——第 3 章會從零實作。切換三種形態對照：

不用重新訓練，就能換任務：zero-shot 與 few-shot

GPT 這型模型雖然只被訓練來「接話」，卻展現出驚人的彈性——不改一根權重就能做各種新任務。用書中圖 1.6 的三個例子切換看：

名詞澄清（transformer ≠ LLM）：不是所有 transformer 都是 LLM——transformer 也能做電腦視覺；也不是所有 LLM 都用 transformer——有基於 RNN、CNN 的路線（動機是計算效率，能否追上還待觀察）。本書中「LLM」一詞專指 GPT 這類 transformer-based 模型。

1.6

A closer look at the GPT architecture（GPT 架構）

input：前文
"This is an"→GPT＝只留 decoder單向（左→右）· GPT-3 疊 96 層→output：下一個字 "example"
→ 接回輸入，再來一輪 ↻

GPT 的出身：OpenAI 的 Radford 等人在論文《Improving Language Understanding by Generative Pre-Training》中首次提出；GPT-3 是同一套想法的放大版（更多參數、更大資料，2020 年發表）。而 ChatGPT 初版背後的模型，就是把 GPT-3 用 InstructGPT 論文的方法在大型指令資料集上微調而來——1.3 那條「預訓練 → 微調」流程的真實案例。

相比 1.4 的原始 transformer（encoder 與 decoder 各重複 6 次），GPT 把架構簡化成只留 decoder，但疊得深得多：GPT-3 有 96 層 transformer 層、共 175B 參數。訓練任務就只有 next-word prediction——一種自監督（self-supervised）的「自我標註」：用句子裡的下一個字當標籤，標籤「隨讀隨生」。

自回歸：自己的輸出，餵給自己

decoder 一次只生成一個字，生完就把它接回輸入再算下一個——所以 GPT 這型模型稱為 autoregressive（自回歸）模型。按「下一步」走一遍書中圖 1.8 的三輪迭代（機率為示意值）：

這就是 ChatGPT 逐字吐字的原因——不是打字特效，模型真的「一次只算一個 token」。也因為每個新字都基於前面所有字（包括自己剛生成的），輸出才會前後連貫。
🔰 emergent behavior（湧現行為）

GPT 的訓練任務只有「接下一個字」，沒有人教它翻譯——但它會翻譯（還會拼字修正、分類、摘要）。這種「沒被明確訓練、卻自己出現」的能力叫emergent behavior：模型接觸大量多語資料後的自然結果。不必為每個任務養一個模型，「一個模型、多樣任務」——LLM 的價值主張正是來自這裡。順帶一提：更新的架構（如 Meta 的 Llama）仍是同一套概念加上小改動——把 GPT 學透，就拿到了看懂現代 LLM 的鑰匙。

本章小結

  • LLM 是什麼：在海量文字上以 next-word prediction 訓練的深度神經網路；「large」指參數量與資料量兩種規模。座標：AI ⊃ ML ⊃ DL ⊃ LLM，且因會生成而屬於 GenAI。
  • 預訓練是自監督：標籤就是文字裡的下一個字（右移一格），不需人工標註——所以數兆詞的原始文字都能當教材。
  • 兩步訓練：先在無標註大語料上預訓練成 foundation model，再用小的標註資料微調——instruction fine-tuning（→ 助理）與 classification fine-tuning（→ 分類器）兩大類。
  • transformer（2017，為翻譯而生）＝encoder（讀懂輸入）＋ decoder（逐字生成）；核心是 attention——生成每個字時能選擇性回看整段輸入。
  • BERT vs GPT：BERT 取 encoder 做克漏字（masked word prediction）、擅長理解；GPT 取 decoder 做接龍、擅長生成。GPT-3：96 層、175B 參數、訓練 ~300B tokens、成本估計 $4.6M。
  • 自回歸與湧現：GPT 一次生成一個 token、輸出接回輸入（autoregressive）；zero-shot／few-shot 與翻譯等能力是 emergent behavior。
  • 三階段藍圖：ch2–4 打地基（資料、attention、架構）→ ch5 預訓練＋載入開源權重 → ch6–7 微調成分類器與助理。
