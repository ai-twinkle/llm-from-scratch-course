本章定位：還缺哪些積木？

attention 只是引擎，要組成一台能吐字的 GPT 還需要一批「看似配角、缺一不可」的零件。這是本章的積木清單——它會隨你往下讀自動打勾（也可以點擊跳到該節）：

🔰 先建立心態

GPT 的架構沒有想像中複雜。它的「深」不是來自千奇百怪的結構，而是同一個 transformer block 重複 12 次。理解一個 block，就理解了整台模型的主體。另外：本章做完的模型還不會講人話——權重是隨機的，訓練（pretraining）是第 5 章的事，所以結尾生成出亂碼是預期結果。

🔰 先分清楚兩種模式：訓練 vs 推論（本頁會隨時標註）

模型有兩種「使用模式」，千萬別混在一起：🏋️ 訓練 training＝forward 算預測 → 算 loss → backward 算梯度 → 更新權重，靠大量資料把權重調好（第 5 章的主題）；💬 推論 inference＝權重固定不動、只做 forward，一次吐一個字。本章主要在搭兩種模式共用的架構；但凡講到「梯度」（LayerNorm 的動機、shortcut、前向×反向）都是在講訓練，而 4.7 的文字生成是推論。各節標題旁會掛上對應的標籤。

暖身：tensor 就是「多層的 array」

模型看不懂文字，只吃數字。一個字（token）→ 一串數字（vector）；一句話 → 一疊向量（matrix）；一批句子 → 再疊一層（3 維 tensor）。以單字 moves 為例：

# 一個 token：一條向量（GPT-2 用 768 個數字描述一個字；以下數值為示意，非書中輸出） "moves" → token ID 6100 → tok_emb[6100] = [ 0.0189, -0.4471, 0.7671, …, 0.1029 ] # 768 個數字 # 一句話（4 個 token）：一個 2 維矩陣，形狀 [4, 768] "Every effort moves you" → [[ …768 個… ], # Every [ …768 個… ], # effort [ …768 個… ], # moves [ …768 個… ]] # you # 一批（2 句）：3 維 tensor，形狀 [2, 4, 768] ← 之後看到方括號數字就是在說這個

「參數（parameters）」就是模型裡可被訓練調整的權重（weights）。一層 2048×2048 的權重矩陣就有 2048×2048 = 4,194,304 個參數。我們瞄準 GPT-2 small（約 124M 參數）：它的預訓練權重公開、筆電跑得動；GPT-3 架構幾乎相同，只是規模放大到 175B。

佔位骨架 DummyGPTModel

在刻好每個零件前，先用「什麼都不做、原樣回傳」的假模組把骨架搭起來，看清楚資料怎麼從輸入流到輸出：

class DummyGPTModel(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])   # token 查表成向量
        self.pos_emb = nn.Embedding(cfg["context_length"], cfg["emb_dim"]) # 位置查表成向量
        self.drop_emb = nn.Dropout(cfg["drop_rate"])
        self.trf_blocks = nn.Sequential(                                  # 12 個（假的）block
            *[DummyTransformerBlock(cfg) for _ in range(cfg["n_layers"])])
        self.final_norm = DummyLayerNorm(cfg["emb_dim"])                  # （假的）最終正規化
        self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)

    def forward(self, in_idx):
        tok_embeds = self.tok_emb(in_idx)
        pos_embeds = self.pos_emb(torch.arange(in_idx.shape[1], device=in_idx.device))
        x = tok_embeds + pos_embeds     # 「是什麼字」+「在第幾位」
        x = self.drop_emb(x)
        x = self.trf_blocks(x)
        x = self.final_norm(x)
        return self.out_head(x)         # 投影回 50257 維 → logits

👆 點一個維度
關鍵：輸入幾個 token，輸出就有幾個對應向量——模型對每一個位置都給出「下一個 token 是誰」的 50257 個分數（logits，尚未過 softmax）。生成時我們只取最後一個位置的那條。
常見疑問：那前面位置的 logits 要拿去 argmax 嗎？💬 推論時不用——前面位置的字已經固定（它們就是輸入本身），只對最後一格做 argmax，早期位置的 logits 直接丟棄。那幹嘛算它們？① transformer 是一次矩陣運算算完所有位置，各位置 logits 是同一次計算順便掉出來的；② 真正的主場在🏋️ 訓練：每個位置都是一筆「預測下一個字」的訓練樣本——4 個 token 一次 forward 就得到 4 筆監督訊號（第 5 章的 loss 正是對所有位置一起算），這是 GPT 訓練效率高的關鍵。
4.2

Normalizing activations with layer normalization

input：x
[2, 4, 768]→LayerNorm(768)每個 token 的 768 維 → mean 0 / var 1，再乘 scale 加 shift→output：x′
[2, 4, 768]（形狀不變）

深層網路常遇到 vanishing / exploding gradient（梯度消失／爆炸），訓練極不穩定。Layer normalization 的核心想法：把一層的輸出調整成 mean 0、variance 1（unit variance），加速收斂、穩定訓練。

🔰 白話說明

像考試「調分」：把一組高低不齊的數字拉成「平均 0、散開程度一致」的標準形狀。它是每個樣本自己調自己的（沿特徵維），跟 batch 裡其他樣本無關——這是它和 batch normalization 最大的差別，也是它不怕 batch size 變動、適合 LLM 的原因。

互動①：訊號穿過 30 層，會發生什麼事？

把一個訊號丟進 30 層的網路（真實矩陣相乘）。第 l 顆點＝訊號過完第 l 層時的大小——精確說是該層輸出向量的 RMS（均方根：√mean(x²)，24 個數字「典型有多大」；不用平均值是因為正負會抵銷）。點縮小到看不見＝訊號消失、點變紅＝數值爆掉；滑鼠停在點上可看實際數值。

🔰 s 是什麼？（不是 activation，也不是單一權重）

模擬中每層算的是 y = (s·W)·x：W 是那一層的權重矩陣（幾百個不同的數字，每層都不一樣），s（scale）是把整個 W 一起放大／縮小的倍率——代表該層權重的「整體大小」。點的大小才是 activation（訊號值）：s 決定 activation 每過一層被放大或縮小多少，走到第 l 層時量級 ≈ s^l。可怕之處在於要連乘 30 次：s=0.9 看似無害，0.9³⁰ ≈ 0.04；s=1.1 也看似無害，1.1³⁰ ≈ 17.4。真實網路裡沒有一顆叫 s 的旋鈕——權重大小是初始化與訓練自然形成的，沒人能保證它剛好停在 1.0（Xavier/He initialization 就是想把每層的等效 s 調到 ≈ 1）。

互動②：那 loss 會怎樣？（瀏覽器裡真的訓練 300 步）🏋️ 訓練

同一個 6 層小網路，只差在初始權重的倍率 s（就是互動①那顆旋鈕，訓練開始時權重的整體大小）：s=0.3（太小）、s=3.0（太大）、s=3.0 但每層加 LayerNorm。在你的瀏覽器裡實際跑 300 步訓練（真實 forward + backward + 權重更新），把 loss 畫出來：

把結論記成三句話：梯度 → 0 ⇒ 權重不再更新 ⇒ loss 卡住不動（注意：不是變 0！）；梯度 → ∞ ⇒ 數值溢位 ⇒ loss 變 NaN，訓練直接炸掉；加 LayerNorm ⇒ 每層訊號被校準回同一量級 ⇒ loss 才能順利下降。這正是 GPT 在每個子層前都放 LayerNorm（Pre-LN）的原因。

先看 dim 往哪個方向算

書中的例子：torch.randn(2,5) 過一層 Linear(5,6)+ReLU，得到一個 2×6 的輸出（ReLU 把負值壓成 0，所以沒有負數）。切換 dim，看平均值是「沿哪個方向」算的：

一步步做正規化（數值即時計算，和書中輸出一致）

class LayerNorm(nn.Module):
    def __init__(self, emb_dim):
        super().__init__()
        self.eps = 1e-5                                  # 防止除以零
        self.scale = nn.Parameter(torch.ones(emb_dim))   # 可訓練縮放
        self.shift = nn.Parameter(torch.zeros(emb_dim))  # 可訓練平移

    def forward(self, x):
        mean = x.mean(dim=-1, keepdim=True)              # keepdim 保形狀好做廣播
        var  = x.var(dim=-1, keepdim=True, unbiased=False)
        norm_x = (x - mean) / torch.sqrt(var + self.eps)
        return self.scale * norm_x + self.shift
unbiased=False：為什麼分母用 n 而不是 n−1？統計學上從「樣本」估計母體變異數時，慣例會把分母改成 n−1（稱為 Bessel's correction）：因為用樣本自己的平均當中心，會系統性低估散佈程度，除以 n−1 可以修正這個偏差；反之分母用 n，得到的就是所謂「有偏（biased）」的變異數估計。這裡刻意選 n，理由有二：① 對 LLM 來說這個 n 是 embedding 維度（768），n 和 n−1 的差別約 0.13%，實務上可忽略；② 相容性才是重點——原始 GPT-2 是用 TensorFlow 實作的，其變異數預設就是除以 n；我們採一模一樣的算法，之後把 OpenAI 的預訓練權重載進來時，LayerNorm 的行為才會和那些權重當初被訓練時完全一致。
4.3

Implementing a feed forward network with GELU activations

input：x
[2, 3, 768]→FeedForwardLinear 768→3072 · GELU · Linear 3072→768→output
[2, 3, 768]（形狀不變）

LLM 不用傳統 ReLU，改用更平滑的 GELU（Gaussian error linear unit）。精確定義是 GELU(x) = x·Φ(x)（Φ 是標準常態的 CDF），實務用便宜的近似式（原始 GPT-2 也用它）：

GELU(x) ≈ 0.5·x·(1 + tanh[ √(2/π)·(x + 0.044715·x³) ])
滑鼠移到曲線上逐點比較。ReLU 在 0 有尖角、負值一律歸零（梯度也歸零，神經元可能「死掉」）；GELU 平滑、負值仍有小的非零輸出，訓練更好調。

互動：GELU 的好處——神經元不會「死掉」

🔰 先懂情境

一個神經元收到的輸入值 z（＝前一層輸出乘上權重的加總）在訓練過程會不斷漂移——因為權重每一步都在變。下圖把 24 個神經元照它們的 z 值排在數線上，z = 0 是 ReLU 的生死線：一旦漂到左邊（負區），ReLU 的輸出是 0、梯度也恰為 0 → 權重從此不再更新 → 這個神經元永遠困在負區（dying ReLU）。GELU 在負區仍保留一點梯度，還能被訓練慢慢拉回右邊。

FeedForward：768 → 3072 → 768

transformer block 裡的前饋網路＝兩個 Linear 夾一個 GELU。先放大 4 倍到更寬的空間探索特徵、過非線性、再收縮回原維度。點「下一步」看形狀流動（用書中的輸入 [2, 3, 768]）：

class FeedForward(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.layers = nn.Sequential(
            nn.Linear(cfg["emb_dim"], 4 * cfg["emb_dim"]),  # 768 → 3072（放大 4 倍）
            GELU(),
            nn.Linear(4 * cfg["emb_dim"], cfg["emb_dim"]),  # 3072 → 768（收縮回來）
        )
為什麼進出同維度？進去 768、出來也 768，這種一致性才能把模組一路往上疊，中間不必調整維度——這是 transformer「可堆疊」的關鍵設計。

4.4

Adding shortcut connections（加入捷徑連接）🏋️ 為了訓練

input：x→任意子層Linear+GELU / attention / FFN …→＋x 走捷徑
直接加回→output = 子層(x) + x
（形狀相同才能相加）

Shortcut connection（又稱 skip / residual connection，源自電腦視覺的 ResNet）作法樸素到不可思議：把某一層的輸入，直接加到那一層的輸出上。這等於替反向傳播的梯度開一條「抄近路」的旁路，繞過中間運算、避免梯度層層縮小。

書中用一個 5 層小網路（每層 Linear+GELU）實測。forward 裡可選擇要不要加捷徑：

def forward(self, x):
    for layer in self.layers:
        layer_output = layer(x)
        if self.use_shortcut and x.shape == layer_output.shape:
            x = x + layer_output      # 加回輸入＝捷徑
        else:
            x = layer_output
    return x

4.5

Connecting attention and linear layers in a transformer block

input：x
[2, 4, 768]→TransformerBlockLN → attention → +x　·　LN → FFN → +x→output
[2, 4, 768]（內容重新編碼）

把 MultiHeadAttention、LayerNorm、FeedForward、dropout、shortcut 全部組進一個 transformer block——GPT-2 small 把它重複 12 次。兩個子模組分工明確：

  • multi-head attention：「跨位置」的資訊交流——讓每個 token 看到並吸收其他 token 的資訊。
  • feed forward：「逐位置」的加工——對每個位置的向量獨立做非線性變換。

Pre-LayerNorm vs Post-LayerNorm：本實作把 LayerNorm 放在進子層之前（Pre-LN，現代 GPT 的做法，訓練較穩定）；原始 transformer 論文放在子層之後（Post-LN），較容易訓練不穩。
形狀不變 ≠ 內容不變：輸入 [2, 4, 768]、輸出仍是 [2, 4, 768]，但每個位置的向量已被重新編碼成整合了整句資訊的 context vector——每過一個 block，token 的表徵就「更懂上下文」。
4.6

Coding the GPT model（實作完整 GPT）

input：token IDs
[batch, tokens]→GPTModeltok+pos embedding → dropout → 12×TransformerBlock → final LN → linear head→output：logits
[batch, tokens, 50257]

把骨架裡的假模組換成真貨，就是完整的 GPTModel：embedding → 12× TransformerBlock → final LayerNorm → linear output head。輸出形狀依然 [2, 4, 50257]——複習一下這三個數字：2＝batch（一次餵了 2 句話）、4＝每句 4 個 token、50257＝每個 token 位置對詞彙表全部 50257 個字各給一個分數（logits）。也就是「2 句 × 每句 4 個位置 × 每個位置一整排『下一個字是誰』的評分」（回 4.1 的維度互動）。

self.trf_blocks = nn.Sequential(
    *[TransformerBlock(cfg) for _ in range(cfg["n_layers"])])   # 這次是真的 12 層
self.final_norm = LayerNorm(cfg["emb_dim"])
self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)

參數量：163M 與 124M 的差別（weight tying）

total_params = sum(p.numel() for p in model.parameters()) # Total number of parameters: 163,009,536

祕密在 weight tying（權重綁定）：原始 GPT-2 讓輸出層直接重用 token embedding 的權重。看形狀就懂——兩層一模一樣、互為反向操作：

tok_emb.weight → [50257, 768]
token ID → 768 維向量
≠
out_head.weight → [50257, 768]
768 維向量 → 50257 個分數
163,009,536 − (50257 × 768) = 163,009,536 − 38,597,376 = 124,412,160 # ≈ 124M ✓

我們的實作採「兩層各自獨立」（163M），因為經驗上訓練效果較好、現代 LLM 也多如此；tying 的好處是省記憶體。163M 個 32-bit float（每個 4 bytes）合計 621.83 MB——「小」模型的權重也相當可觀。

🔰 weight tying 為什麼「合理」，不只是省參數

兩層做的事互為鏡像：token embedding 把「字」查成「向量」；輸出層把「向量」變回「每個字的分數」。若共用同一份權重，第 t 個字的 logit 就等於 h · e_t——拿目前的表徵 h 去和每個字的 embedding 做內積（相似度）：「h 長得最像哪個字的 embedding，哪個字分數就最高」。這在語意上非常自然：同一套「字的座標系」同時負責讀入與輸出。

  • 省多少：50257 × 768 ≈ 38.6M 個參數（float32 約 147 MB）——tying 後 163M → 124M、權重檔 622 MB → 約 475 MB。
  • 代價：輸入與輸出被迫共用同一套表徵，少了一份可以分工的權重。實務上兩派都有：原始 GPT-2 綁（省參數）；本書實作不綁——經驗上各自獨立的訓練表現通常略好。
  • 之後會再遇到：載入 OpenAI 的 GPT-2 預訓練權重時，因為官方是綁的，要把同一份 embedding 權重同時填進 tok_emb 與 out_head。

Generating text（生成文字）💬 推論

input：idx
[1, n]→model → 取 [:, -1, :] → softmax → argmax只看最後一個位置的預測→output：idx
[1, n+1]（接回再跑，迴圈）

生成式模型一次產生一個 token、接龍式寫句子。單步機制：模型輸出 logits → 只取最後一個位置那條 50257 維向量 → softmax 轉機率 → argmax 挑最大 → 把新 token ID 接回輸入 → 再跑一輪。

def generate_text_simple(model, idx, max_new_tokens, context_size):
    for _ in range(max_new_tokens):
        idx_cond = idx[:, -context_size:]        # 超長就裁掉前面
        with torch.no_grad():                    # 推論不用算梯度，省時省記憶體
            logits = model(idx_cond)
        logits = logits[:, -1, :]                # (batch, n_token, vocab) → 只取最後一格
        probas = torch.softmax(logits, dim=-1)
        idx_next = torch.argmax(probas, dim=-1, keepdim=True)  # greedy：挑機率最高
        idx = torch.cat((idx, idx_next), dim=1)  # 接回序列尾巴
    return idx
softmax 其實可以省：softmax 是單調函數，不改變大小順序——logits 最大值的位置＝softmax 後最大值的位置，直接對 logits 做 argmax 結果相同。這裡保留它，是為了把「logits → 機率」的過程完整展示。

本章小結

  • Layer normalization 沿特徵維（dim=-1）把輸出調成 mean 0 / var 1，穩定訓練；scale/shift 可訓練、eps 防除零、unbiased=False 相容 GPT-2 權重。
  • GELU 平滑、負值保留非零梯度，比 ReLU 更適合深層 LLM。
  • FeedForward：768 → 3072 → 768，「膨脹—非線性—收縮」探索更豐富的表徵。
  • Shortcut connection 把輸入加回輸出；5 層實測：無捷徑第 1 層梯度 0.0002（消失）、有捷徑 0.22（穩定），差三個數量級。
  • Transformer block＝attention（跨位置）＋ feed forward（逐位置），Pre-LayerNorm＋雙捷徑；形狀不變、內容重新編碼。
  • GPTModel＝embedding ＋ 12×block ＋ final norm ＋ output head；我們 163M，GPT-2 靠 weight tying 記為 124M，權重約 622 MB。
  • 生成＝取最後一格 logits → softmax → argmax（greedy）→ 接回輸入循環；未訓練 → 亂碼。
