在無標註資料上預訓練Pretraining on unlabeled data

第 4 章組好的 GPT 只會輸出亂碼——因為權重還是隨機的。這一章補上靈魂：先定義「生成得好不好」的數值指標（loss），再寫訓練迴圈實際預訓練它，接著用 temperature 與 top-k 控制生成的隨機性，最後把 OpenAI 訓練好的 GPT-2 權重直接載進我們的模型。頁面中的關鍵數值都與書中執行輸出一致（示意值會另行標明）。

本章定位：讓模型「學會講話」

整本書的三階段地圖裡，本章是第二階段：預訓練——把第一階段搭好的架構，變成一個能生成連貫文字的 foundation model：

本章的 7 個步驟——會隨你往下讀自動打勾（也可以點擊跳到該節）：

🔰 先建立心態

預訓練的邏輯一句話就講完：拿一段文字、遮住下一個字，讓模型猜；猜錯就把權重往「更會猜對」的方向調一點；重複幾百萬次。本章所有內容——loss、訓練迴圈、評估——都是在把這句話變成可以執行的程式。而「無標註資料」的妙處是：下一個字本身就是標準答案，不需要人工標記，網路上的任何文字都能拿來訓練。

🔰 複習：訓練 vs 推論（本頁會隨時標註）

🏋️ 訓練 training＝forward 算預測 → 算 loss → backward 算梯度 → 更新權重——本章的主戲；💬 推論 inference＝權重固定、只做 forward、一次吐一個字——5.1.1 的生成回顧與 5.3 的解碼策略屬於這邊。各節標題旁會掛上對應的標籤。

暖身：loss 是整章的方向盤

訓練要「往好的方向調」，前提是先能量化「現在有多差」。這個數字就是 loss（損失）：愈小＝模型把正確答案的機率估得愈高。先預告本章會看到的三個關鍵數字：

六步算出 cross entropy loss（書中真實數值）

對真正的 GPTModel（詞彙表 50257）做同樣的事。反向傳播需要一個 loss 函數來衡量「差多遠」——按「下一步」逐步推導：

# PyTorch 一行就把六步全包了（先把 batch 維攤平）：
logits_flat  = logits.flatten(0, 1)      # [2, 3, 50257] → [6, 50257]
targets_flat = targets.flatten()         # [2, 3] → [6]
loss = torch.nn.functional.cross_entropy(logits_flat, targets_flat)
# → tensor(10.7940)　✓ 和手算一致
為什麼要取 log？直接最大化機率也行，但機率是很多小數字的連乘，數值上既容易 underflow、優化又難；取 log 把連乘變連加、把 [0,1] 展開到 (−∞, 0]，數學上更好操作。深度學習慣例是「把某個量最小化」，所以再乘 −1——這就是「負平均對數機率」＝ cross entropy。

5.3

Decoding strategies（解碼策略：控制隨機性）💬 推論

input：最後一格 logits
[50257]→top-k 過濾 → ÷temperature → softmax → multinomial 抽樣取代原本的 argmax→output：下一個 token
（有隨機性、但不亂來）

剛訓練好的模型有個問題：generate_text_simple 用 argmax（永遠挑機率最高的字，稱為 greedy decoding），所以同一個開頭永遠生出同一句話——而且容易整段背出訓練集原文。解法：把「挑最大」改成「按機率抽籤」（torch.multinomial），再用兩個旋鈕控制抽籤的行為。

旋鈕①：temperature scaling（把分佈捏尖或壓平）

書中用 9 個字的迷你詞彙表示範。模型看到 "every effort moves you" 之後輸出這排 logits（下方數值），temperature 就是「把 logits 除以 T 再過 softmax」。拖動 T，機率分佈即時重算（真實計算）：

temperature T = 1.0
# next_token_logits（"every effort moves you" 之後，書中數值） [4.51, 0.89, -1.90, 6.75, 1.63, -1.62, -1.89, 6.28, 1.79] # 最大是 forward（index 3）
對照書中的 torch 實測（T=1、seed 123、抽 1000 次）：573 次量級的 forward、343 次 toward、73 次 closer、2 次 inches、其餘 0 次（實際輸出 582/343/73/2）——大致就是機率 × 1000。本頁的抽樣用瀏覽器內建的固定種子亂數，次數會與 torch 略有出入，分佈趨勢相同。

旋鈕②：top-k sampling（把長尾直接剁掉）

高溫讓輸出多樣，但也讓 "every effort moves you pizza" 這種蠢話有 ~4% 的機會被抽中。top-k 的對策：只保留 logits 最高的 k 個字，其他一律設成 −∞——softmax 後機率變 0，永遠抽不到。拖動 k 看四個步驟（k=3 為書中例子）：

every effort moves you ____

組裝成新的 generate()

def generate(model, idx, max_new_tokens, context_size,
             temperature=0.0, top_k=None, eos_id=None):
    for _ in range(max_new_tokens):
        idx_cond = idx[:, -context_size:]
        with torch.no_grad():
            logits = model(idx_cond)[:, -1, :]
        if top_k is not None:                       # ① 先剁長尾
            top_logits, _ = torch.topk(logits, top_k)
            logits = torch.where(logits < top_logits[:, -1],
                                 torch.tensor(float("-inf")).to(logits.device), logits)
        if temperature > 0.0:                       # ② 再調溫、抽籤
            probs = torch.softmax(logits / temperature, dim=-1)
            idx_next = torch.multinomial(probs, num_samples=1)
        else:                                       # temperature=0 → 回到 argmax（greedy）
            idx_next = torch.argmax(logits, dim=-1, keepdim=True)
        if idx_next == eos_id: break                # 遇到結束符提早收工
        idx = torch.cat((idx, idx_next), dim=1)
    return idx
# 用剛訓練好的模型跑（top_k=25, temperature=1.4, seed 123）——書中真實輸出 Output text: Every effort moves you stand to work on surprise, a one of us had gone with random- # 不再逐字背訓練集了（對照 greedy 的輸出："know," was one of the axioms he laid...） # 文法還怪怪的正常——模型只看過 5,145 個 token。
什麼時候用什麼設定？低 T ＋小 k（甚至 greedy）→ 答案穩定、保守，適合事實型任務；高 T ＋大 k → 多樣、有創意但可能離題，適合創作型任務。temperature=0.0 完全關閉抽樣、回到確定性的 argmax。

本章小結

  • Loss＝負平均對數機率（cross entropy）：softmax 取 target 機率 → log → 平均 → 取負；未訓練約 10.98 ≈ ln(50257)。perplexity = exp(loss)，讀作「等效在幾個字之間亂猜」。
  • 資料切分：train 90% / val 10%；val loss 是唯一能誠實回答「有沒有真的學會」的指標。
  • 訓練迴圈八步：epoch → batch → 清梯度 → forward 算 loss → backward → AdamW 更新 → 定期評估 → 生成樣本肉眼檢查。
  • 實測 10 epochs：train 9.78 → 0.39、val 卡在 6.45——小資料多 epoch 必然 overfitting，模型逐字背出《The Verdict》。
  • 解碼策略：multinomial 抽樣取代 argmax；temperature 把分佈捏尖（<1）或壓平（>1）、top-k 把長尾剁掉；T=0 回到 greedy。
  • 存讀權重：state_dict 是「層名 → tensor」字典；續訓要連 AdamW 的 optimizer state 一起存。
  • 載入 OpenAI 權重：config 改回 context 1024、qkv_bias=True；assign() 驗形狀、attention 的 c_attn 要 split 成 Q/K/V、out_head 與 wte 共用（weight tying）。
